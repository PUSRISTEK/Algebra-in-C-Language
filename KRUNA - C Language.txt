C History
Bahasa c diciptakan dan di implementasikan pertama kali oleh Dennis Ritchie di DEC PDP-11
yang menggunakan sistem operasi UNIX. C adalah hasil dari proses pengembangan yang dimulai dari
bahasa yang sudah sangat tua yaitu BCPL. BCPL sebelumnya dikembangkan oleh Martin Richard yang
terinsipirasi dari bahasa B yang diciptakan oleh Ken Thompson. B menjadi penyebab atau awal pengembangan bahasa c di tahun 1970.

Selama beberapa tahun, standar de facto untuk bahasa c adalah versi yang disediakan untuk sistem operasi UNIX.
Ini pertama kali dijelaskan dalam buku The C Programming Language karya Brian Kernighan dan Dennis Ritchie (englewood Cliffs, N.J.: Prentice-Hall, 1978)
Musim panas tahun 1983 sebuah komite memutuskan untuk membuat ANSI (american National Standards Institute) standar yang mampu
menjelaskan bahasa c. proses standarisasi ini berlangsung selama 6 tahun lebih lama dari yang orang orang harapkan.

The ANSI C standar terakhir diadopsi pada december tahun 1989, dan kopi perdananya tersedia pada awal tahun 90an
standar ini juga diadopsi oleh ISO (International Standard Organization) dan menghasilkan sebuah standar yang disebut ANSI/C Standar C.
Pada tahun 1995, amandemen 1 untuk standar c diadopsi, yang didalamnya menambahkan beberapa library. pada tahun 1989 standar untuk c
bersamaan dengan amandemen 1 muncul base document untuk standar C++, menjelaskan c subset untuk c++. versi bahasa c pada tahun 1989
secara umum disebut sebagai c89.

Selama tahun 1990an pengembangan standar bahasa c++ dikonsumsi oleh banyak perhatian programmer. 
begitu juga dengan bahasa c, pengembangan bahasa c terus dilakukan meski dalam waktu yang lama, namun dari pengembangan berhasil menghasilkan standar c yang baru
hasilnya adalah standar c pada tahun 1999 yang disebut c99. secara umum, c99 hampir mempertahankan semua fitur yang ada di dalam bahasa c, dengan demikian c masih c
komite yang mengurus standarisasi c99 fokus kepada dua bidang utama : penambahan beberapa numeric libraries dan pengembangan beberapa penggunaan khusus tetapi sangat inovatif
penambahan fitur baru seperti variable-length arrays dan restrict pointer qualifier. inovasi ini sekali lagi menempatkan c menjadi garis depan pengembangan bahasa pemograman.

C is a Middle Level Language
Pada computer C seringkali disebut bahasa middle level, 
sebagai bahasa yang ada di middle leve c memiliki kemampuan untuk memanipulasi bits, bytes and adresses yang menjadi elemen dasar fungsi komputer.
Semua bahasa pemograman high level mendukung kondep tipe data, tipe data menjelaskan sekumpulan nilai dalam sebuah variabel yang selanjutnya bisa
digunakan untuk melakukan operasi tertentu. Data tipe yang paling umum adalah integer, character, dan floating point. meskipun c memiliki beberapa
data tipe yang tersedia didalamnya dia bukan termasuk strongly tuped language seperti pascal dan ada. Hampir semua konversi tipedata dalam bahasa c bisa dilakukan.
Aspek penting lainya dari bahasa c adalah sedikitnya jumlah keyword yang dimilikinya, sebagai contoh c89 hanya memiliki 32 keyword dan pada c99 hanya ditambahkan 5 keyword.
sementara bahasa pemograman highlevel biasanya memiliki keyword lebih banyak, sebagai perbandingan sebagai contoh pada BASIC didalamnya terdapat 100 keyword.

C is a Structured Language
C adalah bahasa pemograman yang terstruktur.

Compiler versus Interpreter
Compiler membaca seluruh program dan mengkonversinya ke object code, penerjemahan dari souce code ke dalam bentuk yang bisa dieksekusi secara langsung oleh komputer.
object code juga mengacu kepada binary code atau machine code. Secara umum program yang di interpreted menggunakan suatu interpreter berjalan lebih lambat daripada program 
yang dikompil. ingat kembali ini karena compiler mengkonversi program source code kedalam object yang bisa dieksekusi langsung oleh komputer. kompilasi hanya terjadi satu kali,
sementara penterjemahan interpreter dilakukan setiap kali program akan dijalankan. [C - The Complete Reference]


C Language
c digunakan untuk membuat program ukuran kecil dan cepat, bahasa yang paling dekat dengan bahasa mesin.
komputer hanya mengerti satu bahasa yaitu bahasa mesin. sebuah aliran biner (binary stream) yang terdiri dari 0 dan 1. kita akan mengkonversi bahasa c kedalam bahasa mesin dengan bantuan
kompiler. sebuah source code akan dikompil menjadi executable yang didalamnya terdapat bahasa mesin, sekumpulan kode biner 1 dan 0 yang mengalir untuk
dipahami oleh komputer. [lihat lagi headfirst c]
